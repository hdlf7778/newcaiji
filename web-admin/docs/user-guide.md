# 源画像库 — 使用说明

## 一、系统概述

源画像库是一套企业级分布式网页采集管理平台，用于自动化监控，检测更新并采集文章内容。系统每天工作时间（08:00-20:00）每 2 小时自动执行一轮采集，非工作时间每 4 小时一轮。

---

## 二、快速上手（5 分钟）

### 2.1 登录系统
1. 浏览器打开系统地址
2. 输入管理员账号密码，点击「登录」
3. 登录后自动进入监控仪表盘

### 2.2 添加第一个采集源
1. 左侧菜单 → **采集管理** → **采集源列表** → 点击右上角 **「+ 新增采集源」**
2. 填写基本信息：
   - **网站名称**：如「广东省人社厅」
   - **URL**：网站列表页地址，如 `https://hrss.gd.gov.cn/zwgk/`
   - **栏目名称**：如「政务公开」
   - **地区**：如「广东」
3. 点击「下一步」，系统会自动检测网站类型（模板）和生成 CSS 选择器规则
4. 检测完成后进入「试采」步骤，系统会试采 3 篇文章验证规则是否有效
5. 试采通过后点击「提交」，采集源进入待审核状态

### 2.3 审核采集源
1. 左侧菜单 → **审核** → **审核工作台**
2. 查看试采结果（标题、正文预览、字数）
3. 确认无误后按 **Y 键**（或点击「通过」）批准
4. 批准后采集源自动进入生产运行，下一轮调度时开始采集

## 三、各模块详细操作

### 3.1 监控仪表盘
**位置**：左侧菜单 → 监控仪表盘

显示内容：
- **7 个统计卡片**：活跃采集源数、待检测、待审核、异常、今日新增文章、当前成功率、队列积压
- **需要关注区域**：访问异常（红色）、疑似规则失效（黄色）、长期静默（绿色）
- **模板健康度图表**：各模板类型的采集成功率
- **今日采集趋势图**：按时段统计采集数量

> 建议每天上班后先看仪表盘，重点关注红色和黄色告警。

---

### 3.2 采集源管理
**位置**：采集管理 → 采集源列表

#### 查看和筛选
- 顶部 **状态卡片**：点击可快速筛选（活跃/待检测/待审核/异常/暂停/退役）
- **筛选栏**：按模板、平台、地区、健康度组合筛选
- **搜索框**：支持按网站名称、栏目名称、URL 搜索

#### 批量操作
1. 勾选左侧复选框选择多个采集源
2. 底部出现操作栏：**批量审批**、**批量暂停**、**批量退役**

#### 导入采集源
- **批量导入**：上传 CSV/Excel 文件，格式：url, name, column_name, region, template
- **平台导入**：按平台分类导入（如海宁政务云、浙江JCMS等）

#### 采集源状态流转
```
待检测 → 检测中 → 检测完成 → 试采中 → 试采通过 → 待审核 → 已审批 → 活跃
                 ↘ 检测失败              ↘ 试采失败           ↗
                                                    活跃 ↔ 暂停 → 退役
                                                       ↘ 异常
```

---

### 3.3 规则管理
**位置**：采集管理 → 规则管理

每个采集源对应一套规则，规则定义了「如何从网页中提取文章」。

#### 规则内容
- **列表规则 (list_rule)**：定义如何从列表页提取文章链接
  - `list_container`：文章列表容器的 CSS 选择器
  - `list_item`：单条文章的选择器
  - `title_selector`：标题选择器
  - `url_selector`：链接选择器
  - `date_selector`：日期选择器

- **详情规则 (detail_rule)**：定义如何从文章详情页提取正文
  - `content_selector`：正文区域选择器
  - `title_selector`：标题选择器
  - `author_selector`：作者选择器

#### 操作
- **编辑**：点击进入 JSON 编辑器，左侧编辑规则，右侧预览采集效果
- **测试预览**：修改规则后点击「测试预览」，实时查看采集结果
- **保存**：确认无误后点击「保存」，规则版本号自动 +1
- **LLM 重新生成**：如果网站改版导致规则失效，点击此按钮让 AI 重新生成规则

#### 生成方式说明
| 标签 | 含义 |
|------|------|
| LLM生成 | 由豆包大模型自动分析网页结构生成 |
| 手动编写 | 运营人员手动编写的 CSS 选择器 |
| 平台适配 | 针对特定平台（如政务云）的预置规则 |

---

### 3.4 模板列表
**位置**：采集管理 → 模板列表

展示系统支持的 10 种采集模板，每个模板对应一种网站类型：

| 模板 | 说明 | 队列 |
|------|------|------|
| A 静态列表 | 最常见，服务端渲染的 HTML 列表页 | HTTP |
| B iframe加载 | 内容嵌套在 iframe 中 | Browser |
| C API接口 | 直接调用 JSON API | HTTP |
| D 微信公众号 | 微信公众号文章 | HTTP |
| E 搜索监控 | 搜索引擎结果页 | HTTP |
| F 登录态 | 需要登录才能访问 | Browser |
| G SPA渲染 | JavaScript 单页应用 | Browser |
| H RSS订阅 | RSS/Atom feed | HTTP |
| I 政务云 | 政府统一建站平台 | HTTP |
| J 抓包API | 抓包获取的隐藏 API | HTTP |

> HTTP 队列使用轻量级 HTTP 请求，Browser 队列使用 Playwright 浏览器渲染，资源消耗更大。

### 3.5 任务列表
**位置**：采集管理 → 任务列表

#### 统计卡片
- **待处理**：排队等待执行的任务
- **处理中**：正在执行的任务
- **今日成功**：今天成功完成的任务数
- **今日失败**：今天失败的任务数

#### 手动触发采集
1. 点击右上角 **「⚡ 立即采集」**
2. 输入采集源 ID
3. 任务会以最高优先级进入队列，立即执行

#### 任务状态说明
| 状态 | 含义 |
|------|------|
| 待处理 | 已入队等待 Worker 领取 |
| 处理中 | Worker 正在执行采集 |
| 成功 | 采集完成，文章已入库 |
| 部分成功 | 列表页成功但部分详情页失败 |
| 失败 | 采集失败，将自动重试（最多 2 次） |
| 超时 | 执行超时（超过 10 分钟） |
| 死信 | 重试 2 次后仍失败，进入死信队列 |

---

### 3.6 死信队列
**位置**：采集管理 → 死信队列

死信队列存放重试耗尽仍然失败的任务，需要人工介入处理。

#### 处理方式
- **重试**：修复问题后点击「重试」，任务重新进入采集队列
- **忽略**：确认无需处理的任务，点击「忽略」标记
- **批量操作**：勾选多个任务后批量重试或忽略

#### 常见错误类型
| 错误类型 | 含义 | 处理建议 |
|---------|------|---------|
| network_timeout | 网络超时 | 检查目标网站是否可访问 |
| http_403 | 被反爬拦截 | 检查反爬配置，可能需要升级策略 |
| http_429 | 请求频率过高 | 降低采集频率或增加延迟 |
| parse_error | 页面解析失败 | 检查 CSS 选择器规则是否失效 |
| template_mismatch | 模板不匹配 | 重新检测网站模板类型 |
| ssl_error | SSL 证书错误 | 检查目标网站证书 |
| anti_bot_blocked | 反爬拦截 | 升级反爬策略或配置代理 |

---

### 3.7 文章列表
**位置**：内容 → 文章列表

查看所有已采集的文章。

#### 筛选方式
- **来源网站**：按网站名称筛选
- **地区**：按省份筛选
- **发布时间**：按日期范围筛选
- **关键词**：按标题/正文搜索

#### 导出
点击右上角 **「📥 导出 CSV」**，将筛选结果导出为 CSV 文件（UTF-8 编码）。

### 3.8 审核工作台
**位置**：审核 → 审核工作台

审核待上线的采集源，确认试采质量是否达标。

#### 快捷键
| 按键 | 操作 |
|------|------|
| **Y** | 通过（批准该采集源上线） |
| **N** | 跳过（稍后再审） |
| **R** | 拒绝（退回该采集源） |
| **E** | 编辑规则（跳转到规则编辑页） |

#### 审核标准
- 标题完整、无乱码
- 正文内容 > 100 字
- 标题不重复（非模板渲染错误）
- 正文不重复（非同一页面重复采集）

---

### 3.9 Worker 状态
**位置**：系统 → Worker 状态

监控 Python 采集 Worker 的运行状态，每 30 秒自动刷新。

显示信息：
- Worker ID、类型（HTTP/Browser）
- 当前状态和正在执行的任务
- CPU/内存使用率
- 已完成/已失败任务数
- 运行时长和最后心跳时间

> 如果 Worker 超过 60 秒无心跳，会被标记为离线。

---

### 3.10 系统设置
**位置**：系统 → 系统设置

四个配置标签页：

#### 采集调度
- **工作时间段**：默认 08:00-20:00
- **工作时间间隔**：默认 7200 秒（2 小时）
- **非工作时间间隔**：默认 14400 秒（4 小时）
- **手动触发优先级**：默认 100（越大越优先）

#### 自动审批
- 试采评分 ≥ 0.8 时自动通过审批（可配置阈值）

#### 告警配置
- P0 紧急：队列积压 > 20000 或成功率 < 50%
- P1 严重：队列积压 > 10000 或成功率 < 80%
- P2 警告：连续失败 ≥ 5 次

#### 数据保留
- 任务记录保留天数（默认 30 天）
- 日志保留天数（默认 30 天）

## 四、日常运维流程

### 每日巡检（5 分钟）
1. 打开 **监控仪表盘**，检查是否有红色/黄色告警
2. 关注 **成功率** 是否 > 95%
3. 检查 **队列积压** 是否为 0
4. 查看 **死信队列** 是否有新增待处理项

### 异常处理流程

#### 第一步：发现异常

异常来源有三个入口，建议按以下优先级排查：

| 入口 | 查看方式 | 说明 |
|------|---------|------|
| 监控仪表盘 | 红色/黄色告警区域 | 最直观，优先看这里 |
| 死信队列 | 采集管理 → 死信队列 | 重试耗尽的任务，必须人工处理 |
| 任务列表 | 采集管理 → 任务列表 → 筛选「失败」 | 查看正在重试中的失败任务 |

#### 第二步：判断错误类型

在死信队列或任务详情中查看「错误类型」和「错误信息」字段，对照下表判断：

| 错误类型 | 典型错误信息 | 原因 | 紧急程度 |
|---------|------------|------|---------|
| `parse_error` | "列表页提取 0 篇文章" | CSS 选择器规则失效（网站改版） | 高 — 影响该源所有采集 |
| `template_mismatch` | "模板不匹配" | 网站类型变更（如从静态页改为 SPA） | 高 — 需要重新检测模板 |
| `http_403` | "HTTP 403 Forbidden" | 被目标网站反爬拦截 | 中 — 可能影响同域名所有源 |
| `http_429` | "HTTP 429 Too Many Requests" | 请求频率过高被限流 | 中 — 降低频率可恢复 |
| `anti_bot_blocked` | "反爬全层级失败" | 三层反爬策略均被拦截 | 中 — 需升级反爬或配置代理 |
| `network_timeout` | "ConnectTimeout" / "ReadTimeout" | 目标网站不可达或响应慢 | 低 — 通常是临时网络问题 |
| `ssl_error` | "SSL: CERTIFICATE_VERIFY_FAILED" | 目标网站 SSL 证书过期或异常 | 低 — 等网站修复或忽略 |

#### 第三步：按类型处理

---

**情况一：规则失效（parse_error）— 最常见**

表现：任务失败，错误信息为「列表页提取 0 篇文章」或「正文内容为空」。

操作步骤：
1. 在死信队列中记下该任务的**采集源 ID**
2. 进入 **采集管理 → 规则管理** → 找到对应规则 → 点击 **「编辑」**
3. 点击 **「LLM 重新生成」** 按钮，等待 AI 重新分析网页结构
4. 生成完成后点击 **「测试预览」**，检查右侧预览：
   - 是否能提取到文章列表（≥3 篇）
   - 标题和正文是否完整
5. 预览正常 → 点击 **「保存」**
6. 回到 **死信队列** → 选中该任务 → 点击 **「重试」**

可能遇到的问题：
- **LLM 生成的规则仍然不匹配**：先用浏览器打开目标网站，查看网页源代码，确认页面结构是否大幅变化。如果变化很大，可能需要手动编写 CSS 选择器。
- **测试预览提示「请求失败」**：可能是 Python Worker API 未启动，检查 Worker 状态页面。
- **保存后仍然采集失败**：可能是详情规则（detail_rule）也需要更新。在规则编辑器中同时检查列表规则和详情规则。

---

**情况二：网站改版（template_mismatch）**

表现：错误类型为 `template_mismatch`，说明网站技术架构发生变化（例如从静态 HTML 改为 Vue/React SPA）。

操作步骤：
1. 进入 **采集源列表** → 找到该采集源 → 点击进入 **详情页**
2. 点击 **「重新检测」** 按钮，系统会重新访问网站并判断模板类型
3. 检测完成后查看新的模板类型是否变化（例如从 A 静态列表变为 G SPA渲染）
4. 如果模板变更：
   - 系统会自动生成新模板对应的规则
   - 进入规则管理确认新规则 → 测试预览 → 保存
5. 回到采集源列表 → 点击 **「重置」** 将状态恢复为待检测，走完整流程

可能遇到的问题：
- **检测结果和之前一样**：网站可能只是局部改版（改了列表页结构但没换技术框架），这种情况按「规则失效」处理即可。
- **检测为 G SPA渲染但之前是 A 静态列表**：这意味着该源需要从 HTTP Worker 转到 Browser Worker 执行，确认 Browser Worker 正在运行。

---

**情况三：反爬拦截（http_403 / anti_bot_blocked）**

表现：任务失败，HTTP 状态码 403，或错误信息包含「反爬」「Cloudflare」「验证码」。

操作步骤：
1. 先用浏览器手动打开目标网站，确认是否能正常访问：
   - **能正常访问**：说明是采集请求被识别为爬虫，需要升级反爬策略
   - **不能正常访问**（显示维护页/错误页）：说明网站本身有问题，等待恢复
2. 如果需要升级反爬：
   - 进入 **采集源详情** → 查看当前反爬配置
   - 如果当前是 `none`（无反爬），改为 `cookie_auto`（自动 Cookie + UA 轮换）
   - 如果 `cookie_auto` 仍被拦截，升级为 `js_fingerprint`（TLS 指纹模拟）
   - 如果是 Cloudflare 防护的网站，需要使用 Browser Worker（G SPA渲染模板）
3. 配置代理：对于严格限制 IP 的网站，需要在反爬配置中添加代理池地址

可能遇到的问题：
- **同一个域名下多个采集源同时被拦截**：说明该域名有全局反爬策略，需要降低对该域名的请求频率。在反爬配置中增加 `delay_min` 和 `delay_max` 间隔。
- **配置代理后仍被拦截**：可能是代理 IP 已被目标网站封禁，需要更换代理服务商或使用住宅代理。
- **出现验证码**：系统内置了 ddddocr 验证码识别（支持文字/数字/计算题），但如果是 reCAPTCHA/hCaptcha 等复杂验证码，目前无法自动处理，需要标记为「忽略」。

---

**情况四：请求频率过高（http_429）**

表现：HTTP 429 状态码，错误信息「Too Many Requests」。

操作步骤：
1. 进入 **系统设置 → 采集调度**
2. 增大 **工作时间间隔**（例如从 7200 秒改为 10800 秒）
3. 或者只针对特定采集源：进入该采集源的反爬配置，增大 `delay_min`（最小请求间隔）和 `delay_max`（最大请求间隔），例如改为 3-5 秒
4. 等待一段时间后（通常 15-30 分钟），在死信队列中重试

可能遇到的问题：
- **调大间隔后仍然 429**：目标网站可能有更严格的频率限制（如每 IP 每小时 100 次），需要配置多 IP 代理池轮换。
- **只有特定时段出现 429**：可能是网站在高峰期限流更严格，可以将该源的采集时间调整到非高峰期。

---

**情况五：网络超时（network_timeout）**

表现：错误信息「ConnectTimeout」或「ReadTimeout」。

操作步骤：
1. 先确认是单个源还是批量源出现超时：
   - **单个源**：目标网站可能临时故障或响应慢，在死信队列中点击「重试」即可
   - **批量源**（>10 个同时超时）：可能是采集服务器网络问题或 DNS 故障
2. 如果是单个源持续超时：
   - 用浏览器手动打开确认网站是否还在运行
   - 如果网站已关停 → 在采集源列表中将该源 **「退役」**
   - 如果网站正常但响应很慢 → 可能需要增大超时时间（默认连接 8 秒、读取 15 秒）

可能遇到的问题：
- **境外网站经常超时**：国内服务器访问境外网站可能受网络波动影响，建议配置海外代理节点。
- **同一网站时好时坏**：可能是网站服务器负载不稳定，建议将该源的「重试次数」配置保持默认（2 次），系统会自动重试。

---

**情况六：SSL 证书错误（ssl_error）**

表现：错误信息包含「CERTIFICATE_VERIFY_FAILED」或「SSL」。

操作步骤：
1. 用浏览器访问目标网站，查看是否有证书警告
2. 如果浏览器也提示证书错误：
   - 这是目标网站的问题，通常等网站管理员修复即可
   - 在死信队列中标记为 **「忽略」**，等修复后再重试
3. 如果浏览器能正常访问但采集报错：
   - 可能是网站使用了特殊的 SSL 配置，系统已默认忽略证书校验（`verify=False`），一般不会出现此问题
   - 如仍报错，检查是否是代理服务器的 SSL 中间人问题

---

#### 第四步：跟踪验证

处理完异常后，按以下步骤验证：

1. 在死信队列中点击 **「重试」** 重新执行任务
2. 等待 1-2 分钟，到 **任务列表** 中查看该任务状态：
   - **成功** → 问题已修复，关注下一轮是否稳定
   - **仍然失败** → 查看新的错误信息，重复排查流程
3. 建议在修复后连续观察 2-3 个采集周期（4-6 小时），确认稳定后再标记为已解决

#### 批量异常处理技巧

当大量采集源同时出现相同错误时：

1. 在死信队列中按 **「错误类型」** 筛选，确认是否同一类问题
2. 如果是同一个域名/平台的源批量失效：
   - 先修复其中一个源的规则
   - 确认有效后，批量重试同类源
3. 如果是不同域名但相同错误（如批量 timeout）：
   - 检查 Worker 和网络状态
   - 不要一次性批量重试全部，先重试 5-10 个观察效果

### 新增采集源流程
```
新增采集源 → 自动检测模板 → 自动生成规则 → 试采验证 → 人工审核 → 上线运行
```

## 五、常见问题

**Q: 为什么某个采集源一直显示「异常」？**
A: 进入采集源详情查看最近日志，通常是网站改版导致规则失效，需要到规则管理中 LLM 重新生成规则。

**Q: 队列积压过多怎么办？**
A: 检查 Worker 状态页面确认 Worker 都在线运行。如果 Worker 正常但积压仍多，可能需要增加 Worker 实例数。

**Q: 如何暂停某个采集源？**
A: 采集源列表 → 点击网站名称进入详情 → 点击「暂停」按钮。暂停后该源不再参与定时调度。

**Q: 导入的采集源为什么显示「待检测」？**
A: 新导入的采集源需要先完成自动检测（识别模板类型+生成规则），然后试采验证，最后人工审核通过才能上线。
